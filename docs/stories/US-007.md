## Contexto
Permitir que owners convidem usuários via `POST /projects/{id}/invite`.

## Critérios de Aceitação (AC)
1. Owner envia convite para email válido.
2. Convite gera token único com expiração.
3. Convite aceito adiciona usuário ao projeto.
4. Convite inválido/expirado retorna erro.

## Tarefas (Dev)
- Endpoint de convite.
- Envio de email com token.

## Tarefas (QA)
- Unit: geração/validação de token.
- Integration: convite aceito/rejeitado.
- E2E: convite até entrada no projeto.

## Riscos
- Convites reutilizáveis.
- Emails falsos gerando spam.
- Falhas de permissões.
